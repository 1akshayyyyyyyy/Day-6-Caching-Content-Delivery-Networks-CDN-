ðŸ“˜ Core Concept: Caching (CDN)

A **Content Delivery Network (CDN)** places copies of your static files (images, CSS, JavaScript) on **Edge Nodes**â€”servers distributed globally. When a user requests a file, the CDN directs them to the nearest Edge Node.

  • **Cache Hit (Fast):** The Edge Node has the file. Latency is low (e.g., 50ms) because the file travels a short distance.
  • **Cache Miss (Slow):** The Edge Node does *not* have the file. It must travel all the way to the distant **Origin Server** (your main server) to fetch it, resulting in high latency (e.g., 500ms).
